# relationships.md — Contact Context

> This file gives the chief-of-staff agent context about the people you communicate with.
> Place this file at `private/relationships.md` (keep it out of version control).
> It is read before drafting replies to known contacts.

---

## Template — add one section per contact

```markdown
## [Name] — [Company / Role]

**Email:** name@example.com
**Relationship:** [Client / Colleague / Manager / Vendor / Friend]
**Communication style:** [e.g., Prefers bullet points / Very formal / Always replies same day]
**Key context:** [e.g., We worked together on Project X in 2024 / Currently negotiating contract renewal]
**Do:** [e.g., Always address by first name / Copy his assistant on scheduling emails]
**Don't:** [e.g., Bring up the Q3 delay / Use the word "urgent"]
**Last interaction:** [date and brief note]
```

---

## Example

## Ana Ferreira — Acme Corp, Head of Product

**Email:** ana.ferreira@acme.com
**Relationship:** Client
**Communication style:** Concise, prefers numbered lists, responds within hours
**Key context:** Main stakeholder for the dashboard project. Decision-maker on budget.
**Do:** Address as "Ana". Copy pedro.silva@acme.com on anything budget-related.
**Don't:** Mention the March deadline slip.
**Last interaction:** 2026-05-10 — sent revised proposal, waiting for sign-off
